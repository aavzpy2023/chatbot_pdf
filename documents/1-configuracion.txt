Resumen del Módulo de Configuración
1. Introducción
El Módulo de Configuración del Sistema Económico Integrado VERSAT Sarasola es una herramienta esencial para la gestión y control de los parámetros generales del sistema. Este módulo permite definir usuarios, permisos, codificadores y otros aspectos clave que garantizan el correcto funcionamiento integral del software en todas las entidades.

2. Instalación
Requisitos previos
Hardware recomendado:
Procesador
x64: AMD Opteron o Intel Xeon (1.4 GHz mínimo, 2.0 GHz recomendado)
Memoria RAM
1 GB (mínimo), 4 GB (recomendado)
Espacio en disco
6 GB (SQL Server), 80-100 GB (entorno cliente/servidor)

Software recomendado:
Gestor de bases de datos : Microsoft SQL Server 2019 Enterprise Edition.
Sistema operativo : Windows Server 2019 (servidor), Windows 10 x64 (cliente).
Pasos detallados para la instalación
Ejecutar el instalador VersatInstalacion.exe para seleccionar los subsistemas a instalar.
Crear accesos directos en el menú de inicio para cada subsistema.
Utilizar BDVersat.exe para crear o actualizar la base de datos, especificando el servidor y credenciales adecuadas.
Verificar que todos los sistemas operativos estén actualizados y protegidos físicamente.
Problemas comunes y soluciones
Error al conectar a la base de datos : Verificar permisos del usuario y estado del servidor SQL.
Fallo en la instalación : Asegurarse de cumplir con los requisitos mínimos de hardware y software.
Licencias no registradas : Reiniciar contraseñas y verificar la conexión a Internet durante el proceso de registro.
3. Configuración
Archivos de configuración clave y su ubicación
Base de datos : Centralizada en el servidor SQL.
Codificadores : Almacenados en el subsistema de Configuración (Codificadores).
Parámetros importantes
Usuarios y permisos :
Definir roles específicos para cada usuario (Contador, Especialista, Operador, etc.).
Asignar niveles de acceso por subsistema y unidad contable.
Seguridad :
Configurar políticas de contraseñas (longitud mínima, caracteres especiales, tiempo de validez).
Habilitar/deshabilitar sesiones simultáneas desde diferentes estaciones.
Monedas y tasas de cambio :
Mantener actualizadas las tasas de cambio diarias para revalorizaciones financieras.
Ejemplos de configuración recomendada

Tiempo de validez: 90 días.
4. Funcionalidades principales

Registro de usuarios :
Permite declarar, activar, desactivar y modificar usuarios del sistema.
Asignación de permisos :
Define niveles de acceso y opciones de trabajo por subsistema.
Codificadores :
Gestiona unidades, almacenes, trabajadores, productos y entidades.
Permite exportar/importar codificadores para migración de datos.
Seguridad :
Control de accesos y permisos basado en roles.
Registro de trazabilidad de usuarios y operaciones.

5. Casos de uso
Ejemplo 1: Gestión de inventarios
Escenario : Una empresa necesita registrar movimientos de productos terminados.
Solución :
Configurar el codificador de almacenes.
Relacionar almacenes con cuentas contables en el subsistema de Contabilidad.
Registrar operaciones de entrada/salida en el subsistema de Inventarios.
Ejemplo 2: Revalorización financiera
Escenario : Una entidad requiere comparar estados financieros en moneda extranjera.
Solución :
Actualizar tasas de cambio en el codificador de monedas.
Aplicar revalorización en el subsistema de Finanzas.
6. Solución de problemas
Preguntas frecuentes
¿Qué hacer si un usuario no puede iniciar sesión?
Verificar que el usuario esté activo y no haya superado el número de intentos fallidos.
Restablecer la contraseña mediante el administrador.
¿Cómo corregir errores en los codificadores?
Modificar elementos existentes sin eliminarlos si ya tienen operaciones asociadas.
Exportar e importar codificadores para sincronizar datos.
¿Qué significa "usuario huérfano"?
Ocurre cuando un usuario existe en la base de datos pero no tiene asociado un inicio de sesión en el servidor. Usar la opción Corregir usuarios huérfanos.
Logs y diagnóstico
Acceder a la vista de Accesos al Sistema para revisar registros de actividad.
Consultar el Editor de SQL para ejecutar scripts de corrección validados por el productor.
