Resumen del Manual de Versat - Módulo Logística
Introducción
El subsistema de Logística forma parte del software VERSAT Sarasola , diseñado para facilitar el control y gestión de los recursos materiales en almacenes, así como la planificación y ejecución de procesos logísticos. Su objetivo principal es proporcionar herramientas para gestionar solicitudes, órdenes de compra, licitaciones, proyectos y departamentos, asegurando un control eficiente sobre las existencias físicas y estadísticas.

Instalación
Requisitos previos
Sistema operativo : El subsistema requiere un entorno cliente-servidor con sistemas operativos actualizados.
Hardware recomendado :
Servidor: Procesador de 3.0 GHz, 8 GB de RAM, 100 GB de espacio en disco.
Cliente: Procesador de 2.0 GHz, 4 GB de RAM, 80 GB de espacio en disco.
Software requerido :
Gestor de bases de datos: Microsoft SQL Server 2019 Enterprise Edition (recomendado).
Sistema operativo servidor: Windows Server 2019 x64.
Sistema operativo cliente: Windows 10 x64.
Pasos detallados para la instalación
Ejecutar el archivo VersatInstalacion.exe.
Seleccionar la carpeta de instalación y los subsistemas a instalar, incluyendo Logística .
Crear accesos directos en el menú de inicio para cada subsistema.
Problemas comunes y soluciones
Problema : Falta de permisos para acceder a la base de datos.
Solución : Verificar que el usuario tenga acceso restringido al gestor de bases de datos.
Problema : Errores durante la instalación.
Solución : Asegurarse de que el sistema operativo y las dependencias estén actualizadas.
Configuración
Archivos de configuración clave y su ubicación
Los archivos de configuración se encuentran en el subsistema de Configuración .
Parámetros específicos del subsistema de Logística se definen en la actividad Configuración del Subsistema .
Parámetros importantes y su impacto en el rendimiento
Longitud de consecutivos : Define el formato de numeración para órdenes de compra y solicitudes.
Formato de numeración : Ejemplo: P[-]D[-]###### (Proyecto-Departamento-Consecutivo).
Ejemplos de configuración recomendada
Formato de numeración : P[-]D[-]######.
Codificadores : Configurar fases del proyecto, departamentos y proveedores principales antes de iniciar operaciones.
Funcionalidades principales
Descripción de las principales características y su uso
Gestión de proyectos :
Captura, modificación y eliminación de proyectos.
Cambio de fase del proyecto según su estado.
Gestión de departamentos :
Adición y asignación de centros de costo a departamentos.
Control de usuarios por departamento y sus roles.
Solicitudes y órdenes de compra :
Creación de solicitudes de licitación, compras y órdenes de despacho.
Asignación de proveedores principales y posibles proveedores.
Control de inventarios :
Integración con el subsistema de Inventarios para mantener actualizadas las existencias.
Comandos clave y ejemplos de aplicación
Adicionar proyecto :
Código: PROY001
Nombre: Proyecto de Construcción
Fecha de inicio: 01/01/2024
Fase: SL (Solicitud de Licitación)
Cambiar fase de proyecto :
Seleccionar proyecto → Menú contextual → Cambiar fase → Elegir nueva fase.
Adicionar departamento :
Código: DEP001
Nombre: Departamento de Compras
Centro de costo: CC001
Casos de uso
Ejemplos prácticos de implementación en distintos escenarios
Gestión de proyectos industriales :
Crear un proyecto para la fabricación de equipos, definir fases (licitación, compra, producción) y asignar departamentos responsables.
Control de solicitudes de compra :
Generar una solicitud de compra para adquirir materiales necesarios, especificando proveedores y fechas de entrega.
Asignación de roles a usuarios :
Definir permisos para un usuario en el departamento de logística, permitiéndole gestionar solicitudes de compra y autorizar entregas en almacén.
Solución de problemas
Preguntas frecuentes y resolución de errores comunes
Pregunta : ¿Cómo eliminar un proyecto?
Respuesta : Solo se puede eliminar si no está en uso. De lo contrario, desactívelo desde la opción Desactivar .
Pregunta : ¿Qué hacer si una solicitud no se puede modificar?
Respuesta : Verifique que el estado de la solicitud no sea "Confirmado" o "Cerrado".
Logs y diagnóstico de problemas
Los registros de errores se encuentran en el subsistema de Configuración , en la opción Registro de Salvas y Restauraciones .
Utilice la función Buscar en cada vista para localizar errores específicos.
