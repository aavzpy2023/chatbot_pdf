Resumen del Manual de Versat Sarasola - Módulo "Auditoría"
Introducción
Descripción general : El subsistema de Auditoría del software Versat Sarasola es una herramienta informática diseñada para analizar y controlar el origen y destino de recursos humanos, materiales y servicios manipulados por una entidad en un período determinado. Este módulo permite a los auditores revisar integralmente una base de datos sin necesidad de acceder a cada subsistema individual (Figura 1, página 5).
Propósito y contexto de uso : Es útil tanto para entidades que utilizan el software como para organismos externos encargados de auditar sistemas empresariales, entidades presupuestadas y otras modalidades en el país.
Instalación
Requisitos previos :
Sistema operativo compatible: Detallado en el Manual de Configuración.
Dependencias: Base de datos configurada, acceso a otros subsistemas relacionados.
Hardware recomendado: Procesador multicore, al menos 8 GB de RAM, espacio en disco suficiente para almacenar datos transaccionales.
Pasos detallados para la instalación :
Descargar e instalar el paquete del subsistema desde el área comercial de DATAZUCAR.
Registrar el software utilizando las credenciales proporcionadas por el proveedor.
Configurar la conexión con la base de datos central y los subsistemas relacionados.
Realizar pruebas iniciales para verificar la correcta instalación.
Problemas comunes y soluciones :
Fallo en la conexión con la base de datos : Verificar credenciales y permisos de acceso.
Errores de registro : Contactar con el soporte técnico de DATAZUCAR para validar licencias.
Configuración
Archivos de configuración clave y su ubicación :
No requiere configuraciones específicas más allá de la creación de usuarios con permisos adecuados.
Parámetros importantes y su impacto en el rendimiento :
Creación de usuarios con permisos específicos para acceder a la base de datos única del software.
Ejemplos de configuración recomendada :
Usuarios designados por el propio organismo o por organismos especializados, como la Contraloría General de la República.
Funcionalidades principales
Descripción de las principales características y su uso :
Acceso a 15 subsistemas o actividades : Permite auditar operaciones económicas y financieras desde una única interfaz (Figura 3, página 7).
Navegación hasta documentos primarios : Desde cualquier operación registrada, se puede acceder al documento primario que la originó (Figura 2, página 6).
Comandos clave y ejemplos de aplicación :
Selección de período : Permite filtrar operaciones por un rango de fechas específico.
Navegación por clasificadores : Ejemplo: Acceder al historial completo de la cuenta 110-Efectivo en Banco (Figura 1, página 5).
Casos de uso
Ejemplos prácticos de implementación en distintos escenarios :
Auditoría de cuentas bancarias : Revisar operaciones relacionadas con la cuenta 110-Efectivo en Banco, incluyendo liquidaciones de obligaciones.
Auditoría de nómina : Verificar el origen y destino de pagos realizados a empleados.
Auditoría de inventarios : Controlar movimientos de recursos materiales en almacenes.
Solución de problemas
Preguntas frecuentes y resolución de errores comunes :
Falta de acceso a ciertos subsistemas : Verificar permisos de usuario asignados en el subsistema de Configuración.
Inconsistencias en los datos : Asegurarse de que todos los subsistemas estén sincronizados con la base de datos central.
Logs y diagnóstico de problemas :
Utilizar las vistas detalladas del subsistema para identificar inconsistencias en operaciones específicas.
Consultar registros históricos para rastrear el origen de errores.
