Resumen del Manual de Versat Sarasola tambien conocido como  Versat - Módulo "Gestión de la Información"
Introducción
Descripción general : El subsistema "Gestión de la Información" del software Versat Sarasola tambien conocido como  Versat permite visualizar y manipular la información enviada o recibida por cada uno de los subsistemas, siempre que se posean los permisos adecuados. Este módulo es esencial para la gestión de datos entre niveles superiores e inferiores, así como para el procesamiento y seguimiento de la información generada por otros subsistemas.
Propósito y contexto de uso : Facilita el envío, recepción, procesamiento y validación de información crítica para la operación de la entidad, asegurando la integridad y trazabilidad de los datos.
Instalación
Requisitos previos :
Sistema operativo compatible: Detallado en el Manual de Configuración.
Dependencias: Base de datos configurada, acceso a otros subsistemas relacionados.
Hardware recomendado: Procesador multicore, al menos 8 GB de RAM, espacio en disco suficiente para almacenar datos transaccionales.
Pasos detallados para la instalación :
Descargar e instalar el paquete del subsistema desde el área comercial de DATAZUCAR.
Registrar el software utilizando las credenciales proporcionadas por el proveedor.
Configurar la conexión con la base de datos central y los subsistemas relacionados.
Realizar pruebas iniciales para verificar la correcta instalación.
Problemas comunes y soluciones :
Fallo en la conexión con la base de datos : Verificar credenciales y permisos de acceso.
Errores de registro : Contactar con el soporte técnico de DATAZUCAR para validar licencias.
Configuración
Archivos de configuración clave y su ubicación :
Configuración de comunicación: Principal > Datos de la Organización > Página de Comunicación.
Configuración de correo electrónico: Necesaria para habilitar el envío y recepción automática de información.
Parámetros importantes y su impacto en el rendimiento :
Habilitación del uso del correo electrónico (POP3 o IMAP).
Configuración del servidor, usuario, contraseña y cuenta de correo.
Definición de grupos de unidades para la agregación de correos electrónicos.
Ejemplos de configuración recomendada :
Uso del correo predeterminado configurado como POP3 para almacenar mensajes localmente.
Configuración de IMAP para mantener los mensajes en el servidor y evitar pérdida de información.
Funcionalidades principales
Descripción de las principales características y su uso :
Envío y recepción de información : Automatización del envío y recepción de datos entre subsistemas y niveles jerárquicos (Figura 1, página 5).
Estados de la información : Gestión de estados como "En Espera", "Enviada", "Confirmada", "Denegada", etc. (Figura 2, página 6).
Validaciones y denegaciones : Identificación de errores en la importación o falta de paquetes anteriores (Figura 3, página 13).
Comandos clave y ejemplos de aplicación :
Enviar : Envía mensajes pendientes de cada subsistema.
Recibir : Importa información desde el servidor de correos.
Reenviar : Reenvía información seleccionada o confirmaciones/denegaciones.
Procesar : Procesa información recibida en estados como "Denegada" o "Sin Procesar".
Exportar/Importar : Exporta o importa información cuando el correo no está disponible.
Casos de uso
Ejemplos prácticos de implementación en distintos escenarios :
Envío de información contable : Envío de datos de cierre mensual al nivel superior (MINREX) (Figura 4, página 11).
Recepción de nóminas : Importación de datos de nómina desde una misión externa.
Validación de proformas : Aseguramiento de la homogeneidad en los clasificadores de cuentas y centros de costo.
Automatización con servicio : Uso del servicio "ServiceMailVersat Sarasola tambien conocido como  Versat" para recibir y enviar mensajes automáticamente cada cierto tiempo.
Solución de problemas
Preguntas frecuentes y resolución de errores comunes :
Falta de paquete anterior : Reenviar automáticamente el paquete faltante o manualmente si es necesario.
Errores en la importación : Verificar la homogeneidad de clasificadores y cuentas entre bases de datos.
Perdida de información : Configurar correctamente el protocolo IMAP o POP3 según las necesidades.
Logs y diagnóstico de problemas :
Consultar los archivos generados por el servicio "ServiceMailVersat Sarasola tambien conocido como  Versat" para monitorear su funcionamiento.
Utilizar la opción "Salvar Fichero" para reparar archivos adjuntos corruptos antes de reimportarlos.

Resumen del Módulo de Contabilidad General
1. Introducción
El Módulo de Contabilidad General del Sistema Económico Integrado VERSAT Sarasola es una herramienta esencial para el registro, procesamiento y análisis de las operaciones contables en entidades empresariales, presupuestadas y otros tipos de organizaciones. Este módulo permite la automatización del ciclo contable, generación de estados financieros y consolidación de información económica.

Propósito:
Registrar hechos económicos de forma automatizada.
Procesar documentos primarios y generar asientos contables.
Obtener informes financieros como balance de comprobación y estados de rendimiento.
Facilitar la toma de decisiones mediante análisis detallados.
2. Instalación
Requisitos previos
Hardware recomendado:
Procesador
x64: AMD Opteron o Intel Xeon (1.4 GHz mínimo, 2.0 GHz recomendado)
Memoria RAM
1 GB (mínimo), 4 GB (recomendado)
Espacio en disco
6 GB (SQL Server), 80-100 GB (entorno cliente/servidor)
Software recomendado:
Gestor de bases de datos : Microsoft SQL Server 2019 Enterprise Edition.
Sistema operativo : Windows Server 2019 (servidor), Windows 10 x64 (cliente).
Pasos detallados para la instalación
Ejecutar el instalador Versat Sarasola tambien conocido como  VersatInstalacion.exe y seleccionar los subsistemas a instalar (Configuración, Contabilidad, Costos y Procesos son obligatorios).
Crear accesos directos en el menú de inicio para cada subsistema.
Utilizar BDVersat Sarasola tambien conocido como  Versat.exe para crear o actualizar la base de datos, especificando el servidor y credenciales adecuadas.
Verificar que todos los sistemas operativos estén actualizados y protegidos físicamente.
Problemas comunes y soluciones
Error al conectar a la base de datos : Verificar permisos del usuario y estado del servidor SQL.
Fallo en la instalación : Asegurarse de cumplir con los requisitos mínimos de hardware y software.
Licencias no registradas : Reiniciar contraseñas y verificar la conexión a Internet durante el proceso de registro.
3. Configuración
Archivos de configuración clave y su ubicación
Base de datos : Centralizada en el servidor SQL.
Clasificador de cuentas : Almacenado en el subsistema de Contabilidad.
Parámetros importantes
Usuarios y permisos :
Definir roles específicos para cada usuario (Especialista Principal, Especialista en Contabilidad, Contador, etc.).
Asignar niveles de acceso por subsistema y unidad contable.
Seguridad :
Configurar políticas de contraseñas (longitud mínima, caracteres especiales, tiempo de validez).
Habilitar/deshabilitar sesiones simultáneas desde diferentes estaciones.
Monedas y tasas de cambio :
Mantener actualizadas las tasas de cambio diarias para revalorizaciones financieras.

Descripción de características
Registro de comprobantes :

5. Casos de uso
Ejemplo 1: Registro de comprobantes manuales
Escenario : Una entidad necesita registrar ajustes contables al final del período.
Solución :
Acceder al menú Comprobantes de Operaciones .
Capturar el comprobante con los pases correspondientes.
Validar y asentar el comprobante.
Ejemplo 2: Consolidación de operaciones
Escenario : Una empresa requiere consolidar las operaciones de varias unidades contables.
Solución :
Configurar grupos de unidades en el subsistema de Configuración.

Importar y consolidar en el nivel superior.
6. Solución de problemas
Preguntas frecuentes
¿Qué hacer si un usuario no puede iniciar sesión?
Verificar que el usuario esté activo y no haya superado el número de intentos fallidos.
Restablecer la contraseña mediante el administrador.
¿Cómo corregir errores en los comprobantes?
Modificar elementos existentes sin eliminarlos si ya tienen operaciones asociadas.
Exportar e importar comprobantes para sincronizar datos.
¿Qué significa "usuario huérfano"?
Ocurre cuando un usuario existe en la base de datos pero no tiene asociado un inicio de sesión en el servidor. Usar la opción Corregir usuarios huérfanos.
Logs y diagnóstico
Acceder a la vista de Estado de Operaciones para revisar registros de actividad.
Consultar el Editor de SQL para ejecutar scripts de corrección validados por el productor.

Resumen del Manual de Versat Sarasola tambien conocido como  Versat - Módulo Facturación
Introducción
El subsistema de Facturación es uno de los componentes clave del Sistema Integrado VERSAT Sarasola, diseñado para gestionar la emisión de facturas en diversos tipos de operaciones comerciales. Este módulo permite la automatización y transparencia en las operaciones asociadas a la facturación de bienes y servicios, activos fijos, productos de insumo, mercancías para venta y producción terminada. Su integración con otros subsistemas como Inventarios, Activos Fijos y Finanzas garantiza un control eficiente y preciso de las operaciones económicas.

Instalación
Requisitos previos
Sistema operativo : No especificado en el manual, pero se detalla en el manual de Configuración.
Hardware recomendado : No especificado en el manual, pero se detalla en el manual de Configuración.
Software requerido : Subsistemas base (Configuración, Contabilidad General, Inventarios y Activos Fijos).
Pasos detallados para la instalación
Instalar los subsistemas base (Configuración, Contabilidad General, Inventarios y Activos Fijos).
Configurar los parámetros generales en el subsistema de Configuración.
Definir los formatos de numeración para las facturas en el módulo de Codificadores.
Iniciar el procesamiento en Facturación desde el menú Procesos/Inicializar Procesamiento.
Problemas comunes y soluciones
Problema : Errores en la selección de fechas durante el cambio de fecha de procesamiento.
Solución : Asegurarse de que la fecha seleccionada sea igual o posterior a la fecha de procesamiento actual.
Problema : Falta de permisos para acceder al subsistema.
Solución : Asignar roles adecuados en el subsistema de Configuración.
Configuración
Archivos de configuración clave y su ubicación
La configuración general se realiza en el subsistema de Configuración .
Parámetros específicos para facturación se definen en la vista Configuración del Subsistema .
Parámetros importantes y su impacto en el rendimiento
Modificar precio de venta : Permite ajustar precios durante la facturación.
Crear obligación para retenciones : Genera automáticamente obligaciones de pago por retenciones.
Decimales para el precio : Define la precisión de los precios hasta 5 decimales.
Crear obligación en moneda de cobro : Habilita la facturación en una moneda y el registro de cobro en otra.
Ejemplos de configuración recomendada
Conceptos de venta : Configurar cuentas contables para registrar ingresos y costos por moneda.
Impuestos y recargos : Aplicar impuestos sobre ventas y configurar tasas impositivas.
Funcionalidades principales
Descripción de las principales características y su uso
Facturación por servicios :
Registro de ingresos por prestación de servicios.
Generación automática de documentos de obligación en Finanzas.
Facturación de inventarios :
Control de existencias y movimientos en almacenes.
Registro de ventas de insumos, mercancías y producción terminada.
Facturación de activos fijos :
Registro de ventas de activos fijos y su depreciación residual.
Movimientos automáticos en el subsistema de Activos Fijos.
Facturación de bienes y servicios :
Combinación de productos y servicios en una misma factura.
Soporte para diferentes categorías de mercancías y múltiples almacenes.
Comandos clave y ejemplos de aplicación
Convertir prefactura a factura :
Menú contextual: Convertir a Factura.
Anular factura :
Acción: Eliminar documentos generados en Inventarios, Activos Fijos y Finanzas.
Confirmar factura :
Acción: Generar comprobantes contables en los subsistemas vinculados.
Casos de uso
Ejemplos prácticos de implementación en distintos escenarios
Venta de productos terminados :
Utilizar conceptos de contrapartida para registrar ingresos y costos.
Generar documentos de salida en Inventarios.
Facturación de servicios :
Registrar ingresos por servicios utilizando conceptos de Finanzas.
Crear obligaciones de cobro en moneda extranjera.
Control de contratos :
Monitorear la ejecución de contratos por montos y productos/servicios.
Validar restricciones contractuales durante la facturación.
Solución de problemas
Preguntas frecuentes y resolución de errores comunes
Pregunta : ¿Cómo anular una factura confirmada?
Respuesta : Anular los documentos generados en los subsistemas vinculados y registrar la causa de anulación.
Pregunta : ¿Qué hacer si un impuesto no se aplica correctamente?
Solución : Verificar la configuración de impuestos y su asociación con los conceptos de venta.
Logs y diagnóstico de problemas
Los registros de errores se encuentran en la vista Histórico de Facturas .
Utilice la función Ver Documentos Generados para localizar errores específicos en los subsistemas vinculados.

Auditoría de cuentas bancarias : Revisar operaciones relacionadas con la cuenta 110-Efectivo en Banco, incluyendo liquidaciones de obligaciones.
Auditoría de nómina : Verificar el origen y destino de pagos realizados a empleados.
Auditoría de inventarios : Controlar movimientos de recursos materiales en almacenes.
Solución de problemas
Preguntas frecuentes y resolución de errores comunes :
Falta de acceso a ciertos subsistemas : Verificar permisos de usuario asignados en el subsistema de Configuración.
Inconsistencias en los datos : Asegurarse de que todos los subsistemas estén sincronizados con la base de datos central.
Logs y diagnóstico de problemas :
Utilizar las vistas detalladas del subsistema para identificar inconsistencias en operaciones específicas.
Consultar registros históricos para rastrear el origen de errores.
